### US-003: Propagate monitoring from site vote flow to bot API
Status: in_progress
Depends on: US-002

Description:
As a backend maintainer, I want monitoring to be passed to send_embed when available so Discord notifications can reference source monitoring accurately.

Acceptance Criteria:
- [ ] In gmgame-site-backend vote flow, update send_embed request payload to include monitoring when available in vote handler context
- [ ] Existing reward application and balance update logic remains behaviorally unchanged
- [ ] Example: vote handler context with username='Player123' and monitoring='topcraft' results in bot API payload containing both username and monitoring
- [ ] Negative case: when monitoring is absent, backend still sends valid payload and reward flow completes without errors
- [ ] No production config/secret changes are made
